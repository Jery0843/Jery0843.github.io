## 1. Reconnaissance - Surface Discovery

### 1.1 Port Scanning

We begin with a full TCP scan to ensure no hidden services are missed.

```bash
# Aggressive full-range scan with service detection
rustscan -a 10.10.8.1 --ulimit 5000 -r 1-65535 -- -A -sS -Pn -oA browsed
```

### 1.2 Scan Results

```
22/tcp open  ssh     syn-ack OpenSSH 9.6p1 Ubuntu (protocol 2.0)
80/tcp open  http    syn-ack nginx 1.24.0 (Ubuntu)
|_http-title: Browsed
```

#### Observations

- SSH likely requires credentials
- Web server is the main attack surface
- Title hints at browser-related functionality

### 1.3 Host Configuration

During enumeration, multiple virtual hosts are discovered. These must be added locally.

```bash
echo "10.10.8.1 browsed.htb browsedinternals.htb" | sudo tee -a /etc/hosts
```

## 2. Web Enumeration - Chrome Extension Upload Portal

### 2.1 Initial Application Review

```bash
curl -s http://browsed.htb | lynx -dump -
```

### 2.2 Key Findings

- Website allows Chrome extension uploads
- Uploaded extensions are reviewed and manually installed
- Demo extensions provided at `/samples.html`
- Sample extension: `fontify.zip`

This immediately suggests a supply-chain / trust abuse vector.

### 2.3 Sample Extension Download & Analysis

```bash
wget http://10.10.8.1/samples.html -O samples.html
```

Manually download `fontify.zip`, then extract:

```bash
mkdir fontify && cd fontify
unzip ../fontify.zip
tree
```
